# Furor Davidis

> *«Ira sine ratione — bestia est. Ira cum ratione — gladius.»*
> Ярость без разума — зверь. Ярость с разумом — меч.

**Furor Davidis** — Windows-приложение, которое разворачивает AmneziaWG VPN-тоннель и одновременно запускает локальную AI-модель, генерирующую реалистичный маскировочный трафик — делая VPN-соединение невидимым для систем глубокой инспекции пакетов.

Один клик — VPS настроен. Один клик — подключение. Затем разум из 1,7 миллиарда параметров начинает свою тихую работу.

## Имя. Идея.

**Давид и Голиаф** — маленький пастух с пращой против закованного в броню великана.

Современная версия: AI-кластер провайдера — петабайты обучающих данных, тысячи GPU, миллиарды параметров — классифицирует каждый твой пакет за миллисекунды. Это Голиаф.

Давид — модель на 1,7 миллиарда параметров, умещающаяся в 1,1 ГБ оперативной памяти, работающая на твоём процессоре.

Исход кажется предрешённым. Это не так.

**Furor** — это не гнев. Не раздражение. Не импульс. Это особое состояние: разум становится абсолютно чистым, свободным от эмоций, работающим как чистый интеллект — но питаемым яростью как топливом. Воин в этом состоянии не машет руками наугад. Он думает быстрее. Видит точнее. Бьёт туда, где важно.

Маленькая модель не пытается сравняться с Голиафом по масштабу. Она воюет асимметрично — ей не нужно быть больше, ей нужно быть *точной*. Она генерирует именно тот паттерн браузинга, который нейтрализует именно твой DPI, в именно твоей сети, в именно этот час дня. И учится. С каждой сессией она становится чуть лучше в борьбе с именно твоим Голиафом.

*Scrutator frustra laborat.* Инспектор трудится впустую.

## От автора

WireGuard и AmneziaWG — мои рабочие инструменты, не хобби. Я строю каналы связи между людьми и серверами, разбросанными по географии. Я понимаю, как работают сети — не как программист, а как системный инженер, который поднимал серверы с нуля больше раз, чем помнит.

Проблема DPI для меня не теоретическая. Это разница между соединением, которое работает, и тем, которое не работает.

Идея Furor Davidis родилась из простого наблюдения: DPI-системы становятся всё более AI-управляемыми. Они учатся на паттернах. Классический ответ — обфускация — это гонка вооружений, где защитник всегда движется вторым. Я захотел перевернуть это. Вместо того чтобы скрывать тоннель, нужно запутать классификатор. Залить его правдоподобным сигналом. Заставить «подозрительный» трафик раствориться в море легитимно выглядящего HTTPS, следующего реальным паттернам человеческого браузинга.

Локальная модель на клиентской машине означает нулевые внешние зависимости, нулевые API-ключи, ноль данных покидающих твою машину. AI работает на твоём железе, генерирует паттерны для твоей сети, учится на твоих результатах.

Это вайбкодинг — архитектура моя, реализация создана в паре с AI. Полностью открытый исходный код.

```
┌──────────────────────────────────────────────────────┐

│      ↓                                               │

└──────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐

│  UDP → VPS : случайный порт 40000–65000              │
│  junk-обфускация (Jc/Jmin/Jmax/S1/S2/H1–H4)        │
└──────────────────────────────────────────────────────┘
         ↓ на сервере ↓
┌──────────────────────────────────────────────────────┐
│  VPS (Ubuntu)                                        │
│  AWG server endpoint (UDP)                           │
│  xray ДЕКОЙ (TCP:443, dokodemo-door)                 │
│      зондирование → TLS → форвард на DecoyDomain     │
│      НЕ тоннель — только маскировка сервера          │
│  dnsmasq — DNS через тоннель                         │
│  fail2ban — защита SSH от брутфорса                  │
└──────────────────────────────────────────────────────┘
```

### Что видит провайдер

```
Физический адаптер:
  → UDP : 4XXXX      → VPS              (AWG, неизвестный протокол)
  → TCP : 443        → github.com       (cover, настоящий TLS)
  → TCP : 443        → wikipedia.org    (cover, настоящий TLS)
  → TCP : 443        → docs.python.org  (cover, настоящий TLS)

DPI классификатор: 60% обычный веб-браузинг + 40% неизвестный UDP → «допустимо»
```

### Почему cover трафик убедителен

- **Реальные TLS-соединения** к реальным доменам — не симуляция, не воспроизведение. Настоящие HTTPS GET-запросы с полным TLS-хендшейком.
- **Chrome fingerprint** — встроенный TLS Go имеет другой JA3/JA4, чем браузер. Библиотека `utls` (refraction-networking/utls) имитирует Chrome 120 точно.
- **Правильный DNS-путь** — DNS-запросы для cover-доменов уходят через физический адаптер на 1.1.1.1, а не через AWG. Провайдер видит DNS-запрос, за которым следует соответствующий HTTPS — как у настоящего браузера.
- **Сессионные куки** — все запросы сессии используют один cookie jar. Переход с `github.com` на `github.com/search` несёт куки и Referer. Цепочка запросов выглядит как одна пользовательская сессия.
- **Человеческий тайминг** — AI устанавливает `read_sec` для каждого URL исходя из типа контента и расчётного времени чтения. Джиттер ±20%. Никакого паттерна пулемётных запросов.

## AI Движок

Модель выполняет одну конкретную задачу: по поведенческому профилю, текущему состоянию сети и времени суток — сгенерировать реалистичную последовательность URL с временем чтения.

```json
Вход:
{
  "profile": "developer",
  "session_min": 20,
  "sites": ["github.com", "wikipedia.org", "docs.python.org"],
  "rtt_trend": "rising",
  "hour": 14
}

Выход:
[
  {"url": "github.com/golang/go/blob/master/src/net/http/transport.go", "read_sec": 67},
  {"url": "en.wikipedia.org/wiki/Transport_Layer_Security", "read_sec": 94},
  {"url": "docs.python.org/3/library/asyncio.html", "read_sec": 41}
]
```

Модель не принимает решения о тоннеле в реальном времени. Rules Engine (детерминированный код на Go) решает *когда* действовать. Модель решает *как выглядит* сессия браузинга. Разделение ответственности намеренное: LLM прекрасно генерирует правдоподобные человекоподобные последовательности; в роли логики управления реального времени они ненадёжны.

### Редактируемые Cover-списки

| Элемент | Значение |
|---|---|
| Именованные списки | Создание/переименование/удаление списков доменов |
| Активный список в клиенте | У каждого client-профиля свой активный список |
| Синхронизация с HotSwap | Пул decoy-доменов берется из активного AI Cover списка |
| Вместимость | В UI можно хранить длинные списки; AI берет адаптивное подмножество |

## RAG Память — Обучающийся компонент

После каждой cover-сессии приложение измеряет, что произошло с RTT AWG-тоннеля за следующие 5 минут. Если RTT стабилизировался или упал — сессия была эффективной. Если RTT вырос или соединение оборвалось — нет.

Результат оценивается и сохраняется в `furor_memory.json`.

```
Record(контекст, действие)
       ↓
  [через 5 минут]
       ↓
Evaluate(дельта RTT)
  RTT упал   → score +0.3  → outcome: success
  RTT вырос  → score −0.3  → outcome: failure
  без изменений            → outcome: neutral

Score затухает 1%/день → старые данные угасают, недавние победы важнее
Максимум 500 записей → удаляются с наименьшим score
```

Перед генерацией новой сессии AI получает записи с наибольшим score, соответствующие текущему контексту (то же время суток, тот же тип дня, тот же тренд RTT). Модель узнаёт, что браузинг `github.com` в 14:00 в будний день хорошо работает в твоей сети. Она запоминает, что `youtube.com` в 23:00 при растущем RTT срабатывал раньше.

Это не дообучение. Это обучение в контексте — результативные паттерны инжектируются в системный промпт. Модель умнеет применительно к твоей конкретной сети без изменения весов.

`furor_memory.json` переносим. Копируй между машинами.

| **fail2ban** | Защита SSH от брутфорса на VPS (maxretry=5, бан 1 час) |
| **Пост-проверка HotSwap** | Проверка TCP:443 после каждой ротации декой-домена |

## Требования

### Клиент (Windows)

- Windows 10/11, **права Администратора** (обязательно — для инжекции маршрутов)
- Разместить рядом с `FurorDavidis.exe`:

```
FurorDavidis.exe
awg\
    amneziawg.exe
    wintun.dll
```

- Установить **LM Studio** и запустить Local Server на `http://localhost:1234`

### Сервер

- Ubuntu **22.04** или **24.04**
- SSH доступ (root, авторизация по паролю)
- Один открытый UDP порт (случайный, диапазон 40000–65000) — приложение само подберёт
- Docker не нужен — деплой устанавливает всё автоматически

| `Qwen2.5-1.5B-Instruct-Q4_K_M.gguf` | ~1,0 ГБ | Стабильная альтернатива — поиск `Qwen2.5-1.5B Instruct GGUF` |
| `Qwen3-0.6B-Q4_K_M.gguf` | ~400 МБ | Поиск `Qwen3-0.6B GGUF Q4_K_M` — если RAM < 4 ГБ |

> GGUF-репозитории часто меняются — ищи модель по имени напрямую на [huggingface.co/models](https://huggingface.co/models), фильтр по `GGUF`. Важно имя файла для пути в Настройках, а не откуда скачано.

В LM Studio: загрузи модель, открой **Local Server**, запусти API-сервер на порту `1234`.

### 2. Разблокировать файлы

Windows блокирует скачанные exe через Zone.Identifier. Правая кнопка на каждом `.exe` → **Свойства** → **Разблокировать** → ОК. Или в PowerShell:
```powershell
Get-ChildItem -Recurse *.exe | Unblock-File
```

### 3. Развернуть сервер

Вкладка **Deploy**:
- Введи хост VPS, SSH порт, пользователя, пароль
- Настрой декой-домены (по умолчанию: microsoft.com, youtube.com, github.com...)
- Нажми **¡Viva la libertad** — занимает 3–7 минут

Лог стримит все этапы: установка AWG, настройка xray декоя, UFW, dnsmasq, fail2ban, финальная проверка. В начале деплоя выводится маркер `Launching Supremo...`.

### 4. Запустить AI Cover

Вкладка **AI Cover**:
- Проверь/отредактируй активный cover-список
- Нажми **Запустить**

### 5. Подключиться

Вкладка **Connect** → **Libertad**.
После подключения кнопка меняется на **Libero**. При отключении AI Cover останавливается автоматически.

AI-оркестратор начинает свой цикл. Каждые 60 секунд проверяет метрики тоннеля. Если RTT растёт — генерирует cover-сессию и выпускает её через физический адаптер параллельно AWG-тоннелю.

## HotSwap — ротация декой-домена

xray на сервере работает как `dokodemo-door`, форвардящий весь TCP:443 на легитимный домен (декой). Когда DPI-система или наблюдатель зондирует твой VPS на порту 443, они получают валидный TLS-хендшейк с сертификатом `microsoft.com` — потому что они буквально разговаривают с microsoft.com через твой сервер.

**HotSwap** меняет этот декой-домен без перезапуска xray:

```
SSH → sed config.json → docker kill --signal HUP xray
```

xray перечитывает конфиг за ~100мс. Без обрыва соединения. AWG тоннель не трогается.

После каждой смены приложение проверяет, что TCP:443 на VPS всё ещё отвечает. Если нет — в лог пишется предупреждение. Тоннель продолжает работать.

**[AmneziaWG / Команда Amnezia](https://github.com/amnezia-vpn/amneziawg-go)**
Единственный тоннель в этом стеке. Junk-обфускация заголовков, открытый исходный код, активная разработка. Вы сделали WireGuard реально работающим там, где обычный WireGuard блокируется. Этот проект существует потому, что существует ваш.

**[WireGuard / Jason Donenfeld](https://github.com/WireGuard/wireguard-go)**

**[Wails](https://github.com/wailsapp/wails)**

## Манифест

*«Ira sine ratione — bestia est. Ira cum ratione — gladius.»*

DPI-системы, развёртываемые для слежки и ограничения доступа в интернет — большие, дорогие и сложные. В этом и смысл: они созданы побеждать масштабом. Гонка вооружений на таких условиях — большая обфускация против больших классификаторов — это гонка, которую защитник не может выиграть.

Но масштаб имеет слабость. Он вынужден быть общим. Он классифицирует трафик миллионов пользователей по паттернам, обученным на средних значениях. Локальный AI, сражающийся за одно конкретное соединение, в одной конкретной сети, в один конкретный час — это не среднее значение. Это специфическая аномалия, которую общая модель не была обучена поймать.

Давиду не нужно было быть сильнее Голиафа. Ему нужно было быть быстрее, точнее и ударить именно в правильную точку.

Furor Davidis — это та праща. Маленькая, сфокусированная, обучающаяся система — работающая на твоей машине, для твоей сети, несущая твой трафик — против системы, которая не знает о твоём существовании.

*Scrutator frustra laborat.*
